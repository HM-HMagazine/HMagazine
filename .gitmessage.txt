

################
# [<타입>] <내용> 의 형식으로 작성
# issue number가 있으면 slash 뒤에 추가, issue 종료면 'closed' 추가
# 예: [feat/closed #12] 로그인 기능 추가
################
# feat : 새로운 기능 추가
# fix : 버그 수정
# update : 기존 기능 보완
# docs : 문서 추가, 수정
# refact : 기존 코드 개선, 가독성을 위한 변수,클래스명 변경
# style : 로직 변경 없이 코드 포맷, 줄바꿈 등의 수정
# design : 색상, 아이콘 등의 변경
# remove : 필요 없는 코드, 파일 삭제
# test : 테스트 코드 추가, 수정, 삭제
# chore : 위 경우에 포함되지 않는 기타 변경 사항
################
